1

Zainstaluj Chrome DevTools MCP w Claude Code

Chrome DevTools MCP daje Claude'owi kontrolę nad przeglądarką Chrome: agent może otwierać strony, klikać, wypełniać formularze i robić zrzuty ekranu. Nie musisz znać żadnych komend. Wejdź na oficjalną stronę, skopiuj jej adres i poproś Claude'a o instalację:

Otwórz stronę Chrome DevTools MCP →
Panel Claude Code
> zainstaluj mi to MCP: https://developer.chrome.com/blog/chrome-devtools-mcp Enter

Po instalacji zamknij i otwórz VS Code ponownie.

2

Poproś o wypisanie wszystkiego, co to MCP umożliwia

Panel Claude Code
> wypisz mi wszystko, co umożliwia ci Chrome DevTools MCP Enter
3

Przetestuj, czy działa

Panel Claude Code
> otwórz stronę webflow.com i zrób zrzut ekranu Enter

Jeśli otworzy się okno Chrome i Claude opisze Ci, co widzi na zrzucie, działa.

4

Zapoznaj się z obecną wersją strony

W folderze zadania otwórz plik index.html w przeglądarce (prawy przycisk na pliku albo przeciągnij go na okno Chrome).

5

Znajdź inspirację designu

Wejdź na jedną ze stron zbierających gotowe designy i wybierz taki, który Ci się podoba.

Otwórz Webflow Templates →
6

Zrób zrzut ekranu strony-inspiracji

Najwygodniej wtyczką GoFullPage: robi zrzut całej strony od góry do dołu, nie tylko widocznego kawałka.

Zainstaluj GoFullPage →

Alternatywa: zamiast zrzutu skopiuj style strony (podejrzyj jej kod i CSS) i podaj je Claude'owi jako wzór.

7

Spromptuj Claude'a: nowy design + auto-weryfikacja

Przeciągnij zrzut inspiracji prosto do pola rozmowy i poproś o przeniesienie obecnej zawartości strony do nowego designu. Wskaż, żeby:

  • robił zrzuty ekranu, żeby samodzielnie się weryfikować,
  • odwzorowanie designu było pixel-perfect.
Panel Claude Code · przykładowy prompt
> przerób design strony index.html na taki jak na załączonym zrzucie. Cała obecna zawartość i teksty zostają. Po każdej zmianie otwórz stronę, zrób zrzut ekranu przez Chrome DevTools i porównuj z inspiracją, aż odwzorowanie będzie pixel-perfect Enter

1

Stwórz subagenta stres testera

Subagent to osobny, wyspecjalizowany agent z własną instrukcją, którego Claude odpala do konkretnej roboty. Najprościej poprosić Claude'a, żeby sam go napisał:

Panel Claude Code
> stwórz w .claude/agents/ subagenta o nazwie stres-tester, który testuje strony w przeglądarce przez Chrome DevTools MCP: wypełnia formularze różnymi danymi (także błędnymi), klika wszystkie linki i przyciski, zmienia rozmiar okna od telefonu po duży monitor i czyta błędy w konsoli. Ma być bezlitosny: próbuje zepsuć wszystko, co się da, i robi naprawdę holistyczny test. Na końcu raportuje listę znalezionych problemów Enter
2

Zajrzyj do pliku subagenta

Otwórz .claude/agents/stres-tester.md: opis roli, kiedy ma być używany, jakie ma narzędzia. Tak wygląda subagent od środka. Możesz poprawić jego instrukcję jak zwykły tekst.

3

Wpuść go na stronę

Panel Claude Code
> przetestuj stronę index.html stres testerem Enter

Podejrzyj jego pracę: subagent krok po kroku klika po stronie, wypełnia formularz i zmienia rozmiar okna.

4

Przeczytaj raport błędów

Ile problemów znalazł? Na tej stronie naprawdę jest co znaleźć :)

5
Bonus

Napraw błędy i puść testera drugi raz

Poproś o naprawienie znalezionych problemów, a potem odpal stres testera jeszcze raz. Powtarzaj, aż raport będzie czysty.
